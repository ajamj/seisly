# v1.1-hardening: Architectural Hardening

**Goal:** Address critical technical debt and architectural gaps identified in v1.0 reviews, focusing on IPC performance, worker reliability, and data safety.

## 1. High-Performance IPC (Wave 1)
- **Problem:** JSON-RPC over stdio is too slow for large seismic arrays.
- **Solution:** Implement Shared Memory or Binary Framing (Arrow IPC) for transferring large arrays between host and worker.
- **Decisions:** Use `shared_memory` crate for cross-process memory segments.

## 2. Worker Hardening (Wave 2)
- **Problem:** Python workers can hang or consume unbounded resources.
- **Solution:** Heartbeats, timeouts, and memory/CPU quotas.
- **Decisions:** Schema negotiation for API versioning between host and plugin.

## 3. Data Safety (Wave 3)
- **Problem:** `memmap2` can cause SIGBUS if the underlying file is truncated or on a flaky network.
- **Solution:** Add structured signal handling or safe wrappers around `memmap2` access.

## 4. Interpretive Integrity (Wave 4)
- **Problem:** Lack of Undo/Redo and transaction safety for interpretation.
- **Solution:** Centralized Command/Undo stack. Explicit transaction wrapping for picking operations.

## 5. Visual Quality (Wave 5)
- **Problem:** Shaded surfaces look faceted.
- **Solution:** Smooth normal averaging for surfaces/horizons.
