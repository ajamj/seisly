## Technical Notes

### Area-Weighted Normal Algorithm

The key insight is that the cross product of two edge vectors gives a vector whose magnitude is proportional to the triangle area:

```
face_normal = (v1 - v0) × (v2 - v0)
|face_normal| = 2 × triangle_area
```

By accumulating these unnormalized face normals and then normalizing the result, we achieve area-weighted averaging naturally:

```rust
// For each triangle:
let face_normal = cross_product(e1, e2);  // Magnitude ∝ area
for vertex in triangle.vertices {
    normals[vertex] += face_normal;  // Larger triangles contribute more
}
// Then normalize all vertex normals
```

### Triangulation for Auto-Tracking

Auto-tracked horizons are now triangulated using Delaunay triangulation:
- Projects 3D points to 2D (XY plane) for triangulation
- Reconstructs 3D mesh with original Z values
- Automatically computes normals via `triangulate_points`
- Gracefully falls back to point cloud if < 3 points

### Clipping and Normals

When a mesh is split by a plane:
- New vertices are created at intersection points
- Both resulting meshes need normals recomputed
- The `split_mesh_by_plane` function now calls `compute_normals()` on both outputs
