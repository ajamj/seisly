# Phase v1.3: Seismic Visualization & Plotting - Context

**Gathered:** 2026-04-09
**Status:** Ready for planning

<domain>
## Phase Boundary

Implement a professional seismic plotting engine with real-time 2D slicing, variable intensity color mapping, and wiggle trace overlays. This includes SEG-Y performance optimization with persistent sidecar indexing and O(1) grid mapping to eliminate import lag.

**Deliverables:**
- Real-time extraction and rendering of seismic slices (Inlines, Crosslines, Time Slices)
- Support for standard seismic colormaps (Blue-White-Red, Gray)
- Optional wiggle-trace overlay for detailed structural interpretation
- Persistent sidecar indexing (.sfidx) for instant SEG-Y loading after initial scan

**Out of scope (separate phases):**
- Stratigraphy, geobody modeling, pre-stack processing (v1.5+)
- Volume processing chains (v1.5)
- Advanced algorithms (v1.8)
- Plugin system maturity (v1.4)

</domain>

<decisions>
## Implementation Decisions

### Seismic Section Rendering
- **D-01:** Use existing `SeismicRenderer` in `seisly_render` — it's already implemented with `seismic.wgsl` shader
- **D-02:** Colormapping done via 1D texture in fragment shader, not CPU-side lookup
- **D-03:** Gain/clip controls are uniform parameters passed to shader, not CPU preprocessing

### Viewport Integration
- **D-04:** Seismic data uploaded to GPU via `egui_wgpu::CallbackResources` — texture caching to avoid per-frame allocations
- **D-05:** Texture updates only when slice index or volume changes, not every frame
- **D-06:** Use existing `ViewportWidget` (670 lines) — extend, don't rewrite

### Wiggle Trace Overlay
- **D-07:** Wiggle traces rendered as separate pass/layer over the color section
- **D-08:** Toggle via UI checkbox in viewport controls (already exists in current UI)
- **D-09:** Line-based renderer (VA/Wiggle pipeline), not texture-based

### Interactive Slicing Controls
- **D-10:** Sliders for Inline, Crossline, and Time slices in Viewport header
- **D-11:** Gain/clip sliders in sidebar (existing pattern)
- **D-12:** State changes trigger `ctx.request_repaint()` — immediate update, no debounce

### SEG-Y Performance Optimization
- **D-13:** Persistent `.sfidx` sidecar index files using `bincode` serialization
- **D-14:** Regular grid detection → O(1) mathematical offset calculation instead of HashMap lookup
- **D-15:** Initial scan moved to background thread with progress via tx/rx channel
- **D-16:** Non-blocking "Scanning..." notification during initial ingest
- **D-17:** Target: Large SEG-Y files (e.g., Kerry3D) load in <1 second after initial scan

### Color Management
- **D-18:** Standard seismic presets: Blue-White-Red, Gray, Rainbow, Wiggle
- **D-19:** Colormaps stored as 1D textures on GPU for shader access

### Error Handling (from v1.3.1 critical fixes)
- **D-20:** All `.unwrap()` in rendering paths replaced with safe error handling
- **D-21:** CRS transforms return `Err(CrsError::NotImplemented)` instead of identity
- **D-22:** SEG-Y parser returns real inline/crossline ranges (not hardcoded `(1, 1)`)

### Claude's Discretion
- Loading skeleton design during texture upload
- Exact spacing and typography for control labels
- Error state visual design (toast vs inline badge)
- Wiggle trace line thickness and scaling defaults
- Specific bincode version and serde derive strategy for .sfidx

</decisions>

<specifics>
## Specific Ideas

- "Like OpenTect's seismic section display but with modern egui responsiveness"
- Texture caching pattern: store in `CallbackResources`, update only on slice change
- Sidecar index should include: `inline_range`, `crossline_range`, `trace_count`, `sample_count`, `is_regular` flag
- Background scan should not block UI — mouse move/window resize must remain responsive
- Reference OpenTect source for algorithmic understanding only (no code copying — GPL risk)

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- `SeismicRenderer` — Already implements `RenderPipeline` with basic + volumetric pipelines
- `seismic.wgsl` — Shader already supports gain/clip uniforms + wiggle traces
- `ViewportWidget` — 670-line egui widget with map/section views, gain/clip controls, wiggle toggle
- `MmappedSegy` — Memory-mapped SEG-Y access via memmap2
- `BrickCache` — LRU brick caching with moka for out-of-core handling
- `ColormapPreset` — Existing colormap presets including "Seismic" preset

### Established Patterns
- `egui_wgpu::Callback` for custom wgpu rendering within egui
- `CallbackResources` for GPU texture caching
- `tx/rx` channels for background thread communication
- `ctx.request_repaint()` for state change notifications
- Undo/Redo via `HistoryManager` (100 steps)
- Theme system with `ThemeManager` and semantic tokens

### Integration Points
- `SeismicVolume` from `seisly_compute` feeds into `SeismicRenderer` in `seisly_render`
- `ViewportWidget` in `seisly_app` connects to rendering via `egui_wgpu::Callback`
- `SegyReader` trait in `seisly_io` provides trace access to `TraceProvider` implementations
- SQLite database in `seisly_storage` for project metadata (not seismic data itself)

</code_context>
