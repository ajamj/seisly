# Plan: v1.3-05 - UI Polish: Status Bar, Slider Values, Label Consistency

## 1. Context and Objective

From the v1.3 UI audit (score 14/24), three quick-fix copywriting and UX issues were identified as top priority. This plan addresses all three in a single work unit since they all touch the same files and are small, independent changes.

**Source:** `.planning/phases/13-seismic-visualization/v1.3-UI-REVIEW.md`

## 2. Technical Strategy

### WU-1314: Remove fake status bar coordinates
- File: `crates/seisly_app/src/app.rs` (~line 788)
- Current: `status_bar_ui.label("X: 250.5  Y: 312.8  Z: 1523m ")`
- Fix: Replace with meaningful info — loaded volume name, CRS, or sample count. If no volume loaded, show `"No data loaded"`.

### WU-1315: Show Gain/Clip slider values
- File: `crates/seisly_app/src/widgets/viewport.rs` (~lines 101-107)
- Current: `ui.add(egui::Slider::new(&mut self.gain, 0.0..=10.0).text("Gain:").show_value(false))`
- Fix: Change `.show_value(false)` to `.show_value(true)`. Add tooltips: `.tooltip("Amplitude gain multiplier")`.

### WU-1316: Consistent slice label text
- File: `crates/seisly_app/src/widgets/viewport.rs` (~line 66)
- Current: `ui.selectable_value(&mut slice_type, SliceType::TimeSlice, "Time")`
- Fix: Change `"Time"` to `"Time Slice"` for consistency with `"Inline"` and `"Crossline"`.
