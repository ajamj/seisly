# Plan: v1.3-04b - CRS Explicit Error Returns

## 1. Context and Objective

**User story:** As a user working with seismic data in different coordinate reference systems, I want to see an explicit error when CRS conversion is not supported SO THAT I know my coordinates may be wrong rather than silently getting incorrect results.

**From CONTEXT.md decisions:**
- D-21: CRS transform fails loudly — return `Err(CrsError::NotImplemented)` instead of identity

**Current state:** `transformer.rs` returns `points.to_vec()` unchanged (silent identity transform). This is a data integrity hazard — users get wrong coordinates with no warning.

**Scope:** Replace identity transform with explicit error. Full PROJ integration is deferred to a future milestone.

## 2. Technical Strategy

- Add `TransformError::NotImplemented { source_crs: String, target_crs: String }` variant
- Return this error from `transform_points()` instead of identity
- Keep `TransformError` as public error type (used by consumers)
- Update all call sites to handle the error gracefully
