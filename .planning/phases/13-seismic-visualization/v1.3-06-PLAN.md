# Plan: v1.3-06 - UI Polish: Colormap Selector

## 1. Context and Objective

From the v1.3 UI audit, the **single biggest gap** is that 6 colormaps are implemented but there is **no UI control** to switch between them. The `ViewportWidget.colormap` field defaults to `ColormapPreset::Seismic` and is never exposed to the user.

**Source:** `.planning/phases/13-seismic-visualization/v1.3-UI-REVIEW.md` — Visuals section, finding #1

## 2. Technical Strategy

### Add Colormap ComboBox to Viewport Toolbar
- Location: `crates/seisly_app/src/widgets/viewport.rs`, in the horizontal toolbar (~line 57-128)
- Widget: `egui::ComboBox` or `egui::ComboBox::from_id_source` with all `ColormapPreset` variants
- Default: `ColormapPreset::Seismic` (current behavior preserved)
- On change: Update `self.colormap` field → trigger `ctx.request_repaint()` → renderer picks up new colormap

### ColormapPreset Display Labels
- Each enum variant needs a user-friendly display label:
  - `Seismic` → "Seismic (BWR)"
  - `Viridis` → "Viridis" (colorblind-safe)
  - `Magma` → "Magma" (colorblind-safe)
  - `Gray` → "Gray"
  - `Rainbow` → "Rainbow" (⚠️ not colorblind-safe)
  - `BlueWhiteRed` → "Blue-White-Red" (alias of Seismic)

### Optional: Tooltip Warnings
- Rainbow colormap: tooltip warns "Not colorblind-safe — consider Viridis or Magma"
- BlueWhiteRed: tooltip notes "Same as Seismic"
