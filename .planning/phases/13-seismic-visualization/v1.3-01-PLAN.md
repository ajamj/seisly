# Plan: v1.3-01 - Seismic Visualization Core

## 1. Context and Objective

**User stories served by this plan:**
- As a geophysicist, I want to view seismic sections with standard colormaps (Blue-White-Red, Gray) SO THAT I can identify amplitude anomalies and structural features.
- As an interpreter, I want wiggle trace overlays on my seismic sections SO THAT I can see individual trace amplitudes alongside the color section for detailed analysis.
- As a data analyst, I want Rainbow and Viridis colormaps in addition to the default SO THAT I can choose the best visualization for my data type. **Note:** Rainbow colormap is not colorblind-safe (problematic for deuteranopia/protanopia affecting ~8% of males). Viridis and Gray are colorblind-safe alternatives. The Rainbow colormap should include a UI warning: "Rainbow colormap is not recommended for colorblind users — consider Viridis or Gray instead."

**From CONTEXT.md decisions:**
- D-01: Extend existing `SeismicRenderer` — don't rewrite
- D-02: Colormapping via 1D texture in fragment shader
- D-03: Gain/clip as uniform parameters to shader
- D-07: Wiggle traces as separate pass/layer
- D-09: Line-based renderer
- D-18: Presets: Blue-White-Red, Gray, Rainbow, Wiggle
- D-19: Colormaps as 1D GPU textures

**Current state:** `SeismicRenderer` has color + wiggle pipelines. `seismic.wgsl` supports gain/clip uniforms and colormap sampling. `ColormapManager` creates 1D textures. Gaps: (a) missing Rainbow preset, (b) wiggle vertex shader hardcodes `500.0` for normalization, (c) no `SliceType` enum for viewport callback dispatch.

## 2. Technical Strategy

- **Colormaps**: Add `Rainbow` and `BlueWhiteRed` (alias or explicit) to `ColormapPreset`. Each produces 256-entry RGBA8 data via `generate_preset_data`.
- **Shader fix**: Replace hardcoded `500.0` in `vs_wiggle` with uniform-based scaling (`uniforms.u_scale`, `uniforms.v_scale`). Formula: `let x = (uv.x - 0.5) * 2.0 * uniforms.u_scale + offset`.
- **SliceType enum**: `enum SliceType { Inline, Crossline, TimeSlice }` — exported from `seisly_render` for `seisly_app` consumption.

**TDD approach:**
- Colormap: Unit tests verify each preset produces 256 RGBA entries with known color values at key positions
- Shader: Pipeline init test verifies compilation; future integration test will verify wiggle output for known input
- SliceType: Compile-time check (cargo check) — no runtime behavior to test

      ```wgsl
      let x_offset = pos.z * uniforms.gain * 0.05;
      let clip_x = (pos.x / uniforms.u_scale) * 2.0 - 1.0 + x_offset;
      let clip_y = 1.0 - (pos.y / uniforms.v_scale) * 2.0;
      ```
    - Verify no remaining `500.0` literal exists in the `vs_wiggle` function body (may exist elsewhere in shader for unrelated purposes)
    - Keep `fn vs_wiggle` signature unchanged: `@vertex fn vs_wiggle(@location(0) pos: vec3f, @location(1) uv: vec2f) -> @builtin(position) vec4f`
    - The `SeismicUniforms` struct already has `u_scale: f32, v_scale: f32` — no struct changes needed
    - **Important:** Do NOT use `uv` variable in the formula — `uv` is the texture coordinate from `vs_main`, but wiggle positions come from `pos` vertex attribute

## 4. Success Criteria

### User-Facing Metrics
- [ ] **Colormap accuracy:** Geophysicist can visually distinguish positive (red) from negative (blue) amplitudes on a test seismic section — verified by rendering a known test volume and confirming color mapping matches standard seismic display conventions
- [ ] **Wiggle clarity:** Interpreter can see individual trace amplitude peaks and troughs overlaid on color section without visual artifacts — verified by manual inspection of wiggle overlay on test data
- [ ] **Rendering stability:** Viewport renders without panic or visual corruption on any valid seismic volume up to 10 GB — verified by pipeline init test + manual smoke test

### Technical Metrics
- [ ] `ColormapPreset` includes `Rainbow` and `BlueWhiteRed` with correct color generation (verified by unit tests)
- [ ] `vs_wiggle` shader function does NOT contain hardcoded `500.0` — uses `uniforms.u_scale`/`v_scale` instead
- [ ] `SliceType` enum with `Inline`, `Crossline`, `TimeSlice` is publicly exported from `seisly_render`
- [ ] All `cargo test -p seisly_render` tests pass (0 failures)
- [ ] `cargo clippy -p seisly_render -- -D warnings` passes with zero warnings

### Failure Criteria
- **If colormap unit tests fail:** Revert colormaps.rs changes — existing presets (Seismic, Viridis, Magma, Gray) remain functional. Log issue for rainbow color algorithm debugging.
- **If shader fix causes compilation error:** Revert seismic.wgsl — existing hardcoded behavior restored. Log issue with shader error message for debugging.
- **If SliceType breaks external crate compilation:** Keep enum internal to seisly_render, export as opaque type. Viewport callback can use integer discriminant instead.

### MVP Priority
**This plan is MVP-enabling.** Without correct colormaps and wiggle rendering, the seismic visualization is incomplete. Colormaps (WU-1301) and SliceType (WU-1303) are prerequisites for all downstream visualization work. Shader fix (WU-1302) is optional for MVP — existing hardcoded scaling works for standard volumes; fix is needed for arbitrary volume dimensions.

### Rollback Strategy
- If colormap tests fail, revert colormaps.rs changes — existing presets remain functional
- If shader fix causes compilation error, revert seismic.wgsl — existing hardcoded behavior restored
