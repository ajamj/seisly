# Phase v1.3-03 Summary: SEG-Y Performance Optimization

## Goal
Eliminate SEG-Y import lag by implementing persistent sidecar indexing with O(1) grid mapping.

## Achievements

### 1. SegyIndex Schema Migration
- Replaced old schema (tuple ranges + HashMap) with flat fields: `inline_min`, `inline_max`, `crossline_min`, `crossline_max`, `trace_byte_offsets`, `inline_stride`, `crossline_stride`, `trace_header_size`, `trace_data_offset`
- `save()`/`load()` methods with bincode serialization
- 100 MB file size guard in `load()` prevents OOM on malformed index files

### 2. scan_trace_headers() Extraction
- Extracted trace header scanning logic from `MmappedSegy::new()` into public method
- Computes unique inline/crossline step sizes to determine grid regularity
- `is_regular` now correctly checks `inline_steps.len() <= 1 && crossline_steps.len() <= 1`

### 3. get_trace_offset() with Overflow-Safe Math
- Uses `checked_sub()` for i32 subtraction (prevents panic on `i32::MIN - i32::MAX`)
- Uses i64 arithmetic for stride multiplication (prevents silent truncation)
- Returns `None` for out-of-range coordinates

### 4. Staleness Check
- Compares SEG-Y file mtime vs `.sfidx` mtime
- Automatically rescans when SEG-Y file is newer than cached index

### 5. parse_metadata Fix
- Reads actual inline (bytes 188-191) and crossline (bytes 192-195) from first trace header
- Falls back to (1,1) only when trace header is unreadable

### 6. Bug Fixes (from adversarial review)
- **BLOCKING FIX 1:** `is_regular` was always true (tautology) — fixed to check step count uniqueness
- **BLOCKING FIX 2:** i32 subtraction panic on overflow — replaced with `checked_sub()`
- **BLOCKING FIX 3:** Silently discarded save error — added `log::warn!` on failure
- **Advisory FIX:** Silent cast truncation (`il_stride as i32`) — replaced with i64 arithmetic
