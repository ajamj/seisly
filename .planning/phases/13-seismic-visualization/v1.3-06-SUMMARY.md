# Phase v1.3-06 Summary: UI Polish — Colormap Selector

## Goal
Add a colormap selector UI to the viewport toolbar so users can switch between all 6 implemented colormaps.

## What Was Accomplished

### WU-1317: Colormap ComboBox added to viewport toolbar
- Added `egui::ComboBox` with all 6 `ColormapPreset` variants in the viewport toolbar, after the Wiggle checkbox
- Selection change triggers `ctx.request_repaint()` to immediately reflect the new colormap
- User-friendly display labels:
  - Seismic (BWR), Viridis, Magma, Gray, Rainbow ⚠️, Blue-White-Red

### WU-1318: Display labels and tooltips
- Added accessibility warning for Rainbow: "⚠️ Not colorblind-safe — consider Viridis or Magma for accessibility"
- Added alias note for Blue-White-Red: "ℹ️ Same as Seismic — explicit alias for clarity"
- All 6 variants selectable and functional

## Verification
- `cargo check -p seisly_app` — PASS (warnings are pre-existing, unrelated to this change)
- Colormap selector appears in viewport toolbar after Wiggle checkbox
- Selection change triggers immediate viewport repaint

## Files Modified
- `crates/seisly_app/src/widgets/viewport.rs` — Added ComboBox, display labels, accessibility tooltips
