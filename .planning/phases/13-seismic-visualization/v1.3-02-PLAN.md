# Plan: v1.3-02 - Viewport Integration & Interactive Controls

## 1. Context and Objective

**User stories:**
- As a geophysicist, I want to navigate through inline, crossline, and time slices SO THAT I can explore the full 3D seismic volume and identify features of interest.
- As an interpreter, I want to toggle wiggle overlay on/off SO THAT I can see individual trace amplitudes when needed without cluttering the color view.
- As a user, I want the viewport to recover gracefully when rendering resources are unavailable SO THAT I do not lose my interpretation work and can continue using other parts of the application.

**From CONTEXT.md decisions:**
- D-04: GPU upload via `egui_wgpu::CallbackResources` — texture caching
- D-05: Texture updates only on slice/volume change
- D-06: Extend existing `ViewportWidget` (670 lines)
- D-08: Toggle wiggle via UI checkbox (already exists as `show_wiggles`)
- D-10: Sliders for Inline/Crossline/Time in Viewport header
- D-12: `ctx.request_repaint()` on state change
- D-20: Replace all `.unwrap()` in rendering paths

**Current state:** ViewportWidget has single hardcoded `section_xline` slider (0..=500). Lacks: inline/time sliders, dynamic ranges, safe unwrap handling.

**Merge conflict note:** This plan modifies `app.rs` (WU-1307). Plan v1.3-03 also modifies `app.rs` (WU-1310 — background scanning, `scanning` bool). This plan is Wave 2 — runs AFTER v1.3-03 completes. **Merge order:** v1.3-03 merges first, then v1.3-02 rebases on main. WU-1307 must adapt to the new `import_seismic` signature from v1.3-03 (which now has background scanning). The modifications are independent (v1.3-03 adds `scanning` state, v1.3-02 adds slice state), but both touch `SeislyApp` struct and `import_seismic` method — careful rebase required.

## 2. Technical Strategy

- **Three-axis slicing**: Replace single `section_xline` with `active_slice_type: SliceType`, `inline_idx`, `crossline_idx`, `time_idx`. Show only relevant slider based on active type.
- **Dynamic ranges**: Read `volume.provider.inline_range()`, `crossline_range()`, `sample_count()`.
- **Fix unwrap panics**: Replace with `.ok()`, `.and_then()`, early returns + `tracing::warn!`.
- **Error UX**: When resources unavailable, show inline error badge "Rendering not ready — loading seismic data..." instead of crashing.

## 4. Success Criteria

### User-Facing Metrics
- [ ] **Navigation efficiency:** Geophysicist can switch between inline, crossline, and time slice views within 2 clicks — verified by manual test: count clicks from default view to each slice type
- [ ] **Wiggle toggle clarity:** Interpreter can instantly see the difference between color-only and color+wiggle views — verified by manual inspection: toggle checkbox, confirm wiggle overlay appears/disappears
- [ ] **Crash-free session:** User can work with seismic data for 30+ minutes without viewport panicking — verified by: load volume, navigate all three slice types, toggle wiggle, verify no panic

### Technical Metrics
- [ ] ViewportWidget has Inline/Crossline/Time selector (radio buttons)
- [ ] Slider ranges dynamic — based on `volume.provider` dimensions
- [ ] All `.unwrap()` removed from prepare(), paint(), sketch handling
- [ ] ViewportCallback dispatches by SliceType
- [ ] `cargo check -p seisly_app` passes
- [ ] `cargo clippy -p seisly_app -- -D warnings` passes

### Failure Criteria
- **If three-axis slicing breaks existing behavior:** Revert WU-1304, fall back to `section_xline` slider — user can still navigate crosslines
- **If unwrap fixes cause regression:** Show inline error badge "Rendering not ready — loading seismic data..." instead of panic. User sees empty viewport with message, not crash.
- **If dynamic slider ranges fail:** Fall back to hardcoded `0..=500` — slider works but may show invalid indices. Log warning.

### MVP Priority
**This plan is MVP-required.** Without interactive slice navigation, users cannot explore 3D volumes. However, unwrap fixes (WU-1305) are higher priority than slicing UI (WU-1304) — stability before features. If time-constrained, ship WU-1305 first, defer WU-1304 to v1.3.1.

### Rollback Strategy
- If unwrap fixes cause regression, revert WU-1305 — existing behavior restored (with crash risk)
- If slicing breaks existing behavior, revert WU-1304 — fallback to `section_xline` slider
