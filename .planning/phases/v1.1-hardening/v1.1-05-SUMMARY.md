## Technical Notes

### Command Pattern Design
- The `Command` trait uses `dyn Any` for the target type, allowing it to work with any state type
- Commands store their own state for undo operations (e.g., the pick being added)
- Undo doesn't remove the pick permanently—it retains the pick data for redo

### Transaction Strategy
- We use deferred transactions (default in rusqlite) for simplicity
- Immediate transactions could be used for write-heavy workloads
- Future enhancement: batch multiple picks into a single transaction for performance

### Memory Management
- Undo stack is limited to 100 operations by default
- Oldest operations are trimmed when the limit is exceeded
- This prevents unbounded memory growth during long sessions

### Future Enhancements
1. **Persistence**: Save undo history to disk for session recovery
2. **Granular Undo**: Allow undoing specific operations by type
3. **Undo History UI**: Show a list of undoable operations
4. **Batch Transactions**: Group rapid picks (e.g., sketching) into single transactions
