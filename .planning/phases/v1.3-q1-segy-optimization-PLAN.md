# Plan: v1.3-Q1 - SEG-Y Performance Optimization

## 1. Context and Objective
Currently, Seisly performs a full O(N) scan of every trace header upon import to build a coordinate index. For large volumes, this causes severe system lag. This plan implements a persistent indexing strategy and mathematical grid mapping to achieve O(1) startup times.

## 2. Technical Strategy
- **Persistent Indexing:** Save the `inline_range`, `crossline_range`, and `trace_offsets` to a `.sfidx` sidecar file using `bincode`. Requires adding `serde` and `bincode` to `seisly_io`.
- **Regular Grid Detection:** Identify if the volume is a standard 3D grid. If so, use mathematical offset calculation instead of a `HashMap` lookup.
- **Background Scan:** Move the initial scan to a background thread with progress communication via the existing `tx/rx` channel in `SeislyApp`.
