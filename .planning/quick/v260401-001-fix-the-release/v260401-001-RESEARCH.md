## Product Positioning

### Brand Identity
- **Name:** Seisly (pronounced: /ˈsaɪzli/)
- **Tagline:** The professional open-source seismic interpretation platform, powered by Rust
- **Target Users:** Geophysicists, geologists, seismic interpreters in oil & gas exploration and research institutions

### Key Value Propositions
1. **High Performance** - Rust-based, GPU-accelerated seismic attribute computation
2. **Modern 3D Visualization** - Real-time rendering with wgpu
3. **Secure Plugin Architecture** - Process-isolated Python workers for AI/ML models
4. **Open Source** - MIT/Apache-2.0 licensed alternative to expensive proprietary tools
5. **Production Ready** - v1.0.0 with professional UI/UX, dockable panels, persistent layouts

### Core Features (v1.0.0)
- Seismic data loading (SEG-Y format) with inline/crossline slicing
- Interactive fault stick picking with RBF surface modeling
- Horizon picking (manual + BFS auto-tracking)
- Real-time 3D visualization of seismic volumes
- Velocity modeling and time-depth conversion
- Well integration (LAS 2.0/3.0 support)
- 20+ seismic attributes (CPU + GPU)
- Python plugin system for deep learning (PyTorch/TensorFlow)

## Recommended Package Descriptions

### Format Guidelines
- **Length:** 80-120 characters (optimized for MSI metadata and release artifacts)
- **Tone:** Professional, technical, concise
- **Structure:** `[Product] - [Core function] for [target audience]`

## cargo-dist / WiX Considerations

### MSI Installer Metadata
Windows Installer has limited field lengths:
- **Product Name:** 64 characters max
- **Manufacturer:** 64 characters max
- **Description:** 256 characters max (but 80-120 is optimal for UI display)

### Recommended WiX Configuration
All descriptions above are compatible with WiX constraints. The proposed format ensures:
- Clean display in Windows Add/Remove Programs
- Professional appearance in GitHub Releases
- Consistent branding across all artifacts
